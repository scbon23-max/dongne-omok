# 캐치마인드 레벨 시스템 1차 설계

## 목표

- 기존 승패형 점수 대신 게임에 기여한 만큼 영구 경험치를 얻는다.
- 레벨은 1부터 100까지이며, 100은 장기 목표로 둔다.
- 순위 보너스는 넣지 않는다. 게임 완료, 정답 속도, 참가자 MVP 선택만 보상한다.
- 게임 종료 후 본인을 제외한 참가자 한 명에게 MVP 투표를 하고, 그 뒤 개인 경험치 결과를 확인한다.
- 게임 종료 후 개인 경험치 화면을 먼저 보여주고, 닫으면 대기 화면으로 돌아간다.
- 레벨 보상은 그림판 테두리, 스티커, 텍스트 이모티콘, 닉네임 박스 등으로 확장한다.

## 경험치 곡선

다음 레벨 필요 경험치는 레벨이 높아질수록 제곱에 가깝게 증가한다.

`다음 레벨 XP = 60 + 18 × 현재 레벨 + 2.4 × 현재 레벨²`

| 도달 레벨 | 누적 경험치 |
| --- | ---: |
| 10 | 2,035 |
| 25 | 18,600 |
| 50 | 122,010 |
| 75 | 385,170 |
| 90 | 650,945 |
| 100 | 883,080 |

99에서 100만 올리는 데 25,365 XP가 필요하다. 한 게임 최대 획득량은 90 XP이며, 보통 30~45 XP를 받는다고 보면 레벨 100은 약 2만~3만 게임 규모의 장기 목표다.

## 한 게임 경험치

| 항목 | 기준 | 상한 |
| --- | --- | ---: |
| 게임 완료 | 정상 종료한 게임 | 15 XP |
| 정답 속도 | 정답을 맞힌 시점마다 2~12 XP | 60 XP |
| MVP | 경기 종료 투표에서 MVP로 선정 | 15 XP |
| 한 게임 상한 | 완료 15 + 정답 60 + MVP 15 | 90 XP |

연습, 미리보기, 1인 게임, 두 라운드 미만으로 끝난 게임에는 경험치를 주지 않는다. 승리와 최종 등수에는 경험치를 붙이지 않는다.

정답 경험치는 각 정답 시점으로 계산한다.

| 정답 시점 | 경험치 |
| --- | ---: |
| 15초 이내 | 12 XP |
| 30초 이내 | 9 XP |
| 50초 이내 | 6 XP |
| 70초 이내 | 4 XP |
| 90초 이내 | 2 XP |

한 게임에서 정답 속도로 얻는 경험치는 최대 60 XP다. 정답자 수, 내 그림을 맞힌 인원, 기존 경기 점수, 최종 순위는 경험치에 사용하지 않는다.

MVP 투표에는 현재 게임의 경험치 지급 대상 참가자만 나온다. 본인은 선택할 수 없고 계정당 한 표만 행사한다. 모든 표가 모이거나 15초가 지나면 가장 많은 표를 받은 한 명이 15 XP를 받는다. 최다 득표자가 동률이면 서버가 그 동점자 중 한 명을 무작위로 선정하며, 한 번 확정된 결과는 바뀌지 않는다.

## 레벨 구간과 닉네임 박스

| 구간 | 이름 | 화면 방향 |
| --- | --- | --- |
| 1~9 | 스케치 | 기본형 |
| 10~24 | 새싹 | 청록 포인트 |
| 25~49 | 컬러 | 색연필 포인트 |
| 50~74 | 작가 | 파란 포인트 |
| 75~89 | 명장 | 주황 포인트 |
| 90~99 | 거장 | 자주 포인트 |
| 100 | 레전드 | 전용 스타일과 효과 |

목업에는 세 가지 안이 들어 있다.

닉네임 박스는 **A형 2줄 구조**로 확정한다. 닉네임을 위에, 레벨을 아래에 표시하고 레벨 구간 색은 아래 레벨 줄에만 사용한다. 여러 명이 한 줄에 있어도 닉네임이 먼저 읽히며, 50레벨부터는 구간별 두 색이 천천히 교차한다.

프레임과 스티커 스타일은 기본 박스 구조를 바꾸지 않고 추후 장착형 보상으로 추가할 수 있다.

## 경험치 결과 표시

경험치 결과는 아래 세 줄만 표시한다.

- `게임 완료` / `+15 XP`
- `정답 속도 · 3회` / `12초 +12 · 27초 +9 · 48초 +6`
- `MVP 보너스` / 선정 시 `+15 XP`, 미선정 시 `+0 XP`

레벨업 시에는 결과 내용을 가리는 전체 오버레이를 띄우지 않는다. 왼쪽 상단 레벨 배지 숫자가 바뀌면서 확대, 링, 색연필 파티클 효과가 재생되고 새 보상은 아래쪽에 이어서 나타난다.

## 저장 구조

- `catchmind_profiles`: 계정별 누적 XP와 현재 장착 보상
- `catchmind_xp_events`: 게임별 XP 지급 기록
- `catchmind_mvp_votes`: 게임별 참가자 한 표
- `catchmind_mvp_results`: 최종 MVP와 추가 지급 경험치
- `catchmind_reward_catalog`: 레벨 보상 목록
- `catchmind_unlocks`: 계정별 해금 보상

`match_id + nickname`을 고유값으로 묶어 같은 게임의 경험치가 두 번 지급되지 않게 한다. 경험치 갱신과 보상 해금은 하나의 데이터베이스 함수에서 함께 처리한다.

브라우저가 보낸 XP 숫자는 그대로 믿지 않는다. 서버 함수가 정답 시점별 경험치를 다시 계산하고 정답 경험치를 60 XP로 제한한다. MVP 보너스는 별도 투표 함수가 참가 여부, 자기 투표, 중복 투표를 검사한 뒤 최종 당선자에게만 지급한다. 다만 현재 게임 진행 자체가 전용 권위 서버에서 돌아가는 구조는 아니므로, 정답 시점 조작까지 완전히 막으려면 추후 서버 판정형 경기 기록을 추가해야 한다.

## 보상 확장 방식

- `board_frame`: 그림판 테두리 한 개 장착
- `nameplate`: 닉네임 박스 한 개 장착
- `sticker`: 최대 네 개 장착
- `text_emote`: 최대 열두 개 장착
- `bundle`: 레벨 100처럼 여러 보상을 한 번에 여는 세트

1~100 모든 레벨에는 보상 칸을 만들어 두고, 아트가 확정되지 않은 레벨은 비어 있는 계획 칸으로 둔다. 보상 추가는 목록에 항목을 넣는 것으로 끝나며 레벨 계산 구조는 바꾸지 않는다.

## 적용 순서

1. 목업에서 경험치 창과 닉네임 박스 안을 선택한다.
2. 실제 게임 종료 결과를 경험치 입력값으로 연결한다.
3. 로컬 데이터 구조와 서버 함수를 검증한다.
4. 별도 승인 후 원격 데이터베이스에 적용한다.
5. 기존 캐치마인드 점수·랭킹 화면을 레벨·성장 화면으로 교체한다.

현재 파일은 로컬 설계와 목업 단계이며 원격 데이터베이스에는 적용하지 않았다.
